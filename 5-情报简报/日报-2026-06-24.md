### ArXiv 论文

**1. SAFARI: Scaling Long Horizon Agentic Fault Attribution via Active Investigation**
http://arxiv.org/abs/2606.24626v1
直接命中的方向。多 agent 系统执行长程任务时，轨迹超长导致现有故障归因方法失效。SAFARI 用主动调查（active investigation）策略来 scaling fault attribution。对跑 agent 工作流的团队有直接参考价值——怎么 debug 那些跑了 20 步才炸的 agent pipeline，至今没有好方案。

**2. Privacy-Preserving RAG via Multi-Agent Semantic Rewriting**
http://arxiv.org/abs/2606.24623v1
多 agent 语义改写做隐私保护 RAG。核心：不让原始敏感文本进检索/生成管道，而是用 agent 改写为语义等价但脱敏的表示。toB 场景（金融、医疗、法务）如果要上 RAG，隐私是绕不开的坎，这篇给的架构可参考。

**3. The Warrant Gap: Claim-Conditioned Re-scoring for Fact-Checking**
http://arxiv.org/abs/2606.24627v1
LLM 做事实核查时经常输出"支持"标签但引用的证据根本不支撑结论。这篇用 claim-conditioned re-scoring 修补这个 gap。对于任何把 LLM 输出当"答案"的 agent 系统都有警示意义——"看起来对"不等于"真的对"。

**4. Themis: Explainable AI for RLHF**
http://arxiv.org/abs/2606.24622v1
RLHF 的可解释性框架。安全对齐方向，如果做 agent 奖励模型或偏好学习，可以关注这个框架提供透明度的思路。

## ⚡ 市场信号 / 搞钱机会

- **Agent 可观测性 = 空白市场。** SAFARI 论文+日常 agent 调试痛点，说明 agent 运行时的 tracing/debugging/fault attribution 工具极度匮乏——这是可以做的产品方向。

- **小模型落地窗口仍在。** FUTO Swipe 用自训练模型做输入法，说明特定场景仍可绕过大模型 API 成本，做本地推理方案。苹果、高通都在推 on-device，时机正好。
